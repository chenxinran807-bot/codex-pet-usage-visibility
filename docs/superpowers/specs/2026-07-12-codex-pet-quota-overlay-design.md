# Codex 桌宠额度常驻面板设计

## 目标

为“像素代码伙伴”桌宠增加一个常驻的精简额度面板，自动显示当前 Codex 账户的 5 小时额度和周额度。每项同时显示剩余百分比与距离重置时间，数据必须来自本机已登录的 Codex，不允许伪造或要求用户再次输入密码。

## 已确认的体验

- 面板一直显示在桌宠旁边。
- 使用紧凑双行布局：
  - `⏱ 5h 72% · 2h18m`
  - `📅 周 41% · 3d9h`
- 面板为深色圆角卡片，不显示进度条。
- 正常状态使用白字；剩余低于 20% 变为橙色，低于 10% 变为红色。
- 点击面板立即刷新，不跳转页面，不触发 Codex 任务或额度消费。
- 桌宠靠近屏幕右边缘时，面板自动移动到左侧；否则默认显示在右侧。

## 产品边界

Codex 自定义宠物包仅包含 `pet.json` 和 `spritesheet.png`，不能运行代码或绘制实时数据。因此本功能由两部分组成：

1. 原桌宠资源包继续负责动画。
2. 一个配套的原生 macOS SwiftUI 小程序负责读取额度并显示透明悬浮面板。

第一版仅支持 macOS，不修改 Codex 应用本体，不注入进程，不抓取网页，不保存账号密码或访问令牌。

## 数据来源

使用本机 Codex App Server 协议：

- 启动时调用 `account/rateLimits/read`。
- 运行时监听 `account/rateLimits/updated`。
- 从 `RateLimitSnapshot` 中读取主时间窗和次时间窗，按窗口时长映射为 5 小时额度与周额度。
- 使用窗口的已使用比例计算剩余比例：`remaining = max(0, 100 - usedPercent)`。
- 使用重置时间戳计算本地倒计时。

该接口目前属于 Codex 实验性本地协议，不是稳定的公开 REST API。应用必须把协议变化视为可恢复错误，并明确显示不可用状态。

## 组件设计

### AppServerClient

负责启动或连接本机 Codex App Server，完成初始化握手、发送额度读取请求、接收额度更新通知和断线重连。该组件不向界面暴露协议细节。

### RateLimitMapper

负责把 App Server 返回的额度快照转换为两个界面模型：5 小时窗口和周窗口。映射优先依据窗口时长与服务端标识，不依赖数组顺序。缺失任一窗口时，该行单独显示不可用。

### QuotaStore

保存当前额度、最后成功更新时间、连接状态和错误状态。启动读取一次，每 60 秒主动刷新一次，同时接受服务端推送。倒计时每分钟在本地更新。

### OverlayWindow

使用透明、无标题栏、置顶但不抢焦点的 macOS 悬浮窗口。窗口只包裹双行卡片，支持点击刷新和屏幕边缘自动换侧。

### PetAnchorTracker

第一版不侵入 Codex 宠物内部状态。它根据桌宠窗口或桌面可见区域确定面板锚点；若无法可靠识别宠物位置，则使用可拖动的独立悬浮面板，并记住用户最后位置。该降级行为必须明确、稳定。

## 数据流

1. 应用启动，连接 App Server。
2. 完成初始化后发送 `account/rateLimits/read`。
3. `RateLimitMapper` 将快照映射为 5 小时和周额度。
4. `QuotaStore` 发布剩余百分比、重置时间和更新时间。
5. SwiftUI 面板刷新两行内容。
6. 后续由 `account/rateLimits/updated` 或 60 秒轮询触发更新。
7. 本地计时器每分钟刷新倒计时文本。

## 异常与降级

- 首次连接失败：指数退避重试 3 次，随后显示两行“暂不可用”。
- 运行中断线：保留最后一次成功数据，并显示弱化的离线标识；后台继续重连。
- 数据结构变化：记录不含隐私的解析错误，界面显示不可用，不猜测额度。
- 仅一个窗口可用：可用行正常展示，缺失行显示 `-- · 暂不可用`。
- 重置时间已过但尚未收到新快照：立即主动刷新；刷新前显示“正在更新”。
- Codex 未登录：显示“请先登录 Codex”，不弹出自建登录框。
- Codex 未运行：面板隐藏；检测到 Codex 重新启动后恢复。

## 隐私与安全

- 不读取或存储用户密码。
- 不复制 Codex 登录凭据到应用自己的存储。
- 仅连接本机回环地址或本机 Unix socket。
- 不开放局域网端口，不上传额度数据。
- 日志不记录令牌、Cookie、完整账户标识或协议原始敏感字段。

## 测试与验收

### 单元测试

- 主/次窗口映射，不依赖返回顺序。
- 已使用比例到剩余比例的换算和边界值。
- 重置倒计时格式：小时、天、过期与时区。
- 20% 和 10% 两个颜色阈值。
- 缺失字段、未知窗口及协议变化。

### 集成测试

- 使用模拟 App Server 验证初始化、读取、通知、断线和重连。
- 验证每 60 秒轮询不会重复创建连接。
- 验证点击刷新只产生读取请求，不产生 Codex 任务。

### UI 验收

- 常驻双行面板不抢键盘焦点。
- 默认显示在桌宠右侧，靠近右边缘时自动移到左侧。
- 5 小时与周额度同时显示剩余比例和重置倒计时。
- 低额度颜色状态正确。
- Codex 关闭时隐藏，重启后自动恢复。
- 读取失败时不显示伪造数字。

## 交付物

- SwiftUI 源码工程。
- 可运行的 macOS 应用包。
- 原“像素代码伙伴”宠物资源包。
- 安装、启动、卸载及故障排查说明。
- App Server 协议兼容性说明。

## 风险

`account/rateLimits/read` 属于实验性本地协议，Codex 更新可能改变方法或字段。实现应把协议客户端和映射逻辑隔离，升级时只需替换这一层。桌宠位置跟踪也可能受 Codex 窗口实现影响，因此第一版必须保留可拖动独立面板的可靠降级路径。
