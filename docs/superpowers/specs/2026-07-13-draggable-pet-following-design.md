# Codex Pet 额度面板跟随与拖动设计

## 目标

额度面板默认跟随 Codex 桌宠。用户可拖动面板调整它相对桌宠的位置；桌宠移动、切换屏幕或应用重启后，面板继续保持该相对位置。双击面板恢复默认贴边位置。

## 当前问题

当前 ChatGPT/Codex 宿主同时暴露一个约 `34×37` 的辅助窗口和一个约 `356×320` 的桌宠窗口。旧筛选规则把桌宠尺寸上限设为 `320×320`，因此排除了真实桌宠，并可能把小辅助窗口纳入候选。面板随后进入固定位置兜底，无法随桌宠移动。

## 行为设计

### 桌宠识别

- 继续只考虑 ChatGPT/Codex 所属、可见、浮层级别的窗口。
- 排除额度面板自身进程和自身 bundle identifier。
- 排除小于 `48×48` 的辅助窗口。
- 接受最大 `400×400` 的桌宠窗口，以覆盖当前 `356×320` 的真实尺寸。
- 存在唯一可信候选时跟随；无法唯一确认时使用屏幕内兜底位置，不猜测锚点。

### 自动跟随

- 首次运行且没有用户偏移时，面板使用现有默认贴边算法，优先放在桌宠右侧，空间不足时翻转到左侧。
- 每次桌宠位置变化时，根据桌宠当前 frame、用户偏移和屏幕可见区域计算面板位置。
- 最终位置必须限制在当前可见屏幕内。

### 自主拖动

- 面板在跟随和兜底状态下都允许直接拖动，不要求命中特定小把手。
- 拖动结束时：若桌宠可用，保存面板原点相对默认跟随原点的偏移；若桌宠不可用，保存绝对兜底位置。
- 保存偏移后，桌宠继续移动时面板保持相同相对偏移，不会被拉回默认位置。
- 拖动期间暂停程序化位置更新，避免鼠标拖动与跟随更新竞争。

### 恢复默认

- 双击面板清除已保存的相对偏移，立即回到默认贴边位置。
- 清除操作不影响额度数据、登录项或桌宠资源。

### 持久化与兼容

- 使用独立的 UserDefaults key 保存相对偏移，不复用旧的绝对兜底位置 key。
- 旧的兜底位置仍只在无法识别桌宠时使用。
- 多屏变化后，保存的偏移仍会经过屏幕边界限制，不允许面板落到不可见区域。

## 组件调整

- `PetAnchorSelection`：更新可信窗口尺寸规则。
- `OverlayPlacement`：支持在默认跟随 frame 上应用相对偏移并进行屏幕限制。
- `OverlayPanel` / `AppKitOverlayPanel`：报告拖动开始、拖动结束和双击事件。
- `OverlayWindowController`：维护当前桌宠 frame、拖动状态和相对偏移；区分程序化移动与用户拖动。
- `OverlayPositionPersistence`：增加相对偏移的读取、保存和清除能力。

## 测试与验收

- `356×320` 桌宠与 `34×37` 辅助窗口同时存在时，只选择桌宠。
- 默认状态随桌宠移动。
- 用户拖动后记录相对偏移，桌宠再次移动时仍保持该偏移。
- 程序化移动不会被误判为用户拖动。
- 拖动期间的跟随更新不会抢夺窗口位置。
- 双击清除偏移并恢复默认贴边。
- 重启控制器后恢复保存的相对偏移。
- 多屏和屏幕边缘场景下始终可见。
- 完整 Swift 测试、真实额度读取、通用 Release 构建和安装安全测试保持通过。
